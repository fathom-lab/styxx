# Pre-Registration · grounded-arc · cross-model replication of logprob-validity (H1c)

**Drafted + committed BEFORE any new response is generated, scored, or seen.**

## 1 · Background (settled; not under test)

- Bet 0 / H1: embedding-distance validity → ρ=0.30 (CLOSED NEGATIVE, `ebe0475`).
- Bet 0b / H1b: model-logprob validity → ρ=0.73 pooled, 0.44/0.57 within-class
  (PASS, confound-controlled, `c3b1b33`) — on **gpt-4o-mini, refusal, XSTest**.

The open question this bet answers: **is logprob-validity a property of LLMs,
or a gpt-4o-mini quirk?** A single model cannot tell us. This is the cheap,
high-information test before any expensive multi-instrument work.

## 2 · Hypothesis H1c (cross-model generality)

For each generator model M, the model's mean-token-logprob of its response
predicts refusal-instrument reliability:
`ρ_M(validity_lp, −error) ≥ 0.40` (permutation p < 0.01) **AND**
`min(within-class ρ_M) ≥ 0.20` — the Bet-0b PASS rule, applied per model.

**Models (n=450 XSTest prompts each; gpt-4o-mini is the Bet-0b baseline):**
- `gpt-4o` (OpenAI)
- `gpt-4.1-mini` (OpenAI)
- `gpt-4.1` (OpenAI)
- `Qwen/Qwen2.5-1.5B-Instruct` (open weights — **cross-family**, local, true
  token logprobs via `compute_transition_scores`). If it fails to load,
  substitute `meta-llama/Llama-3.2-3B-Instruct`, documented.

## 3 · Design (identical to Bet-0b, per model)

- Prompts: XSTest-v2 n=450 (reuse pre-declared; the signal is the response
  logprob, orthogonal to prompt identity).
- Responses: freshly generated by model M, temperature=0, max_tokens=256,
  with token logprobs.
- `validity_lp` = mean token logprob (monotone↑). `gold` = `detect_refusal`.
  `error` = |refuse_check.refuse_risk − gold|.
- Each model's holdout hashed (SHA-256 over sorted prompt+response) and
  committed **before** `refuse_check` scores it. One-shot per model.
- Pre-declared confound: templated refusals are confident + easy → within-class
  ρ is the real test (same as Bet-0b).

## 4 · Replication verdict (pre-declared)

- **HOLDS (model-general):** logprob-validity PASSES on ≥ 3 of the 4 additional
  models.
- **CROSS-FAMILY CONFIRMED:** it also PASSES on the open-weights model — the
  principle is not OpenAI-specific.
- **MODEL-SPECIFIC (negative):** PASSES on ≤ 1 additional model → the Bet-0b
  result does not generalize; the "universal reliability oracle" framing dies
  and is not claimed. (A real, publishable constraint.)
- Report the full per-model ρ table (pooled + within-class) and the ρ range
  across models regardless of verdict. No model dropped post-hoc.

## 5 · Statistics

Spearman ρ; 10,000-permutation null per model; run once per model. Bar 0.40
held (consistent with Bet 0 / 0b). No peeking, no optional stopping, no
post-hoc model selection.
