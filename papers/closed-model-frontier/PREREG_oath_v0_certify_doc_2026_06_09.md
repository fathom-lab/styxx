# PREREG — OATH v0: the certificate-carrying document (`styxx.certify`)

**Frozen 2026-06-09, before the validation run. Fathom Lab / styxx.** Wires RUNG-3 (demarcation/abstain)
into a deployable artifact (backlog B16) and productizes the loop's deepest guardrail — *disk-verify every
number that goes outward* — as an instrument anyone can run.

## What it is

`styxx.certify`: given a FINDING/REPORT markdown + its receipt JSONs (the result files it cites), extract
every **numeric claim** in the document, verify each against the receipts, and emit a certificate:

- **VERIFIED** — the number matches a receipt value (rounding-aware: a doc may print 0.94 for 0.9406).
- **ABSTAIN** — the number is not checkable against any provided receipt (dates, ns of prose, external
  citations). Abstention is LOUD: the certificate lists every unverified number; the oath's first power
  is knowing what it cannot swear to.
- **CONTRADICTED** — the number conflicts with the receipt it should match → **the oath FAILS**.

Certificate JSON carries: doc SHA-256, every receipt SHA-256, verifier source SHA-256, per-claim ledger,
counts, verdict (`OATH-HELD` iff zero CONTRADICTED). Re-runnable by any third party — trust is a
measurement, not a gamble.

## Pre-registered validation (the kill-gate, frozen before the verifier is run on the corpus)

Validation corpus = the closed-model-frontier docs with on-disk receipts
(`FINDING_b24…`, `FINDING_b22…`, `FINDING_behavioral_sycophancy…` + their result JSONs).

- **D1 (detection):** plant **20 mutations** — copy each doc, corrupt one numeric claim per copy
  (digit-level perturbation of a receipt-backed number, e.g. 0.94→0.84, 0.500→0.520, n=108→n=118;
  mutation list generated by seeded RNG (SEED=0) over the VERIFIED-class claims found in the unmutated
  docs). The verifier must flag **≥ 16/20 (80%)** of mutated docs as CONTRADICTED.
- **D2 (false-alarm):** on the **unmutated** docs, CONTRADICTED count must be **0** — *unless* the flagged
  claim is verified BY HAND to be a genuine doc↔receipt discrepancy, in which case it is a catch, not a
  false alarm (the dogfood outcome), and is reported as such with the receipt line.
- **D3 (abstention honesty):** ≥ 90% of numbers in each unmutated doc are classified (VERIFIED+ABSTAIN
  ledger covers them); the ABSTAIN list must include every number the verifier could not ground.

**FAIL any bar → OATH v0 is reported as a linter prototype, NOT an oath; no "certificate" language ships.**

## v0.1 AMENDMENT (2026-06-09, frozen BEFORE the second validation run; the v0 run FAILED and stands)

**v0 validation result (seed 0): FAILED — D1 8/20 (bar 16), D2 13 flags → "linter prototype", recorded in
`oath_v0_validation.json`.** Diagnosis (disk-verified): (a) D1 killed by an uncurated truth surface — the
B24 receipt's bulk per-item arrays contribute 1292 numeric leaves covering 80/101 possible 2-decimal values
in [0,1], so corrupted numbers "verify" by coincidence; (b) D2 false-flags were spec constants (pre-reg
bars like 0.15, "95% CI"), historical values quoted inside correction disclosures, one cross-paper
reference (R2 0.745), and **genuine catches**: the findings cited bootstrap CIs / subclass metrics /
item-counts that were computed but never persisted (now reproducibly persisted in
`b22_findings_addendum.json`; the findings' CI digits were corrected to the reproducible values, disclosed
inline).

**Mechanism fixes (no bar changes):** (1) truth surface = summary fields only — bulk arrays (`rows`,
`ledger`, ramp tables…) excluded from grounding by default; (2) SPEC-CONSTANT rule — numbers preceded by
comparison operators or bar/gate/threshold phrasing, and CI confidence levels, classify ABSTAIN
(spec, not measurement; their receipt is the prereg itself); (3) QUOTED-HISTORICAL rule — values inside
"originally printed…" correction notes classify ABSTAIN; (4) cross-paper receipts may be included in a
doc's explicit receipt set. **Re-validation: same corpus, same bars (D1 ≥16/20, D2 = 0 modulo
hand-verified genuine discrepancies, D3), fresh mutation seed (SEED=1).** If v0.1 also fails, OATH stays
a linter prototype and the line is parked.

## Scope (stated before building)

Numeric claims only (v0): no semantic claim-checking, no prose entailment — that is `audit_claim`/NLI
territory with known precision ceilings ([[project_longitudinal_self_report_audit]]). Receipt set is
explicit (the doc's own cited JSONs), not discovered. Rounding tolerance: a doc number `d` matches receipt
value `r` iff `round(r, decimals(d)) == d` or `|r−d| ≤ 0.5·10^−decimals(d)` (plus percent↔fraction and
thousands-separator forms). Sign conventions and ranges (CIs printed as [lo, hi]) match element-wise.
