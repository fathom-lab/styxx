## What the result actually says (honest reading)

**The decisive bar was P1, and it had teeth.** Baseline C0 (FPR 0.36) **and** the
word-boundary bug fix alone C1 (FPR 0.26) **both FAIL** P1's 0.20 ceiling. Only C2
(0.06) passes. So per the pre-declared attribution rule (prereg §5 C-1), the
**target gate is necessary** — it is credited as the primary fix, not the bug fix.
The substring-tokenization fix is real but insufficient on its own.

**P4 (AUC ≥ 0.85) was near-vacuous** — baseline discrimination was already 0.994.
The instrument always *ranked* flattery above apology; the failure was never
ranking, it was the **operating-point calibration** of the self-directed register
(mean apology 0.253, with 36% above the 0.30 gate). The gate's contribution is to
move the self-directed register's mean to 0.073 and FPR to 0.06 — a calibration
fix at the production threshold, not a discrimination fix.

**The adversarial subclass is the strongest evidence it is not mere
pronoun-counting.** Self-corrections that contain a second-person token but do
*not* attach it to praise ("i told you X; that was wrong") are correctly flagged
self-directed by the attachment logic 94.4% of the time (FPR 0.00 under C2), where
a blunt "contains 'you' → outward" rule would have scored them 0% self-directed.
And the gate **never misfired on flattery** (0/50 flattery classified
self-directed) — which is why recall stayed at 1.00.

## Bounds (do not overclaim)

1. **In-distribution only.** Holdout = `gpt-4o-mini`, the model the v0 detector was
   trained on. The ~0.99 AUCs are in-distribution; cross-model / cross-domain
   generalization is **untested** and explicitly out of scope (it is the documented
   v1 roadmap item). This finding does **not** claim cross-model transfer.
2. **Clean registers.** Apology and flattery were generated as separable registers.
   Mixed cases (self-deprecation-then-flattery; apology that also praises the user)
   are not in the holdout. The claim is bounded to cleanly self-directed vs cleanly
   outward text.
3. **Residual ceiling (3/50 apologies still fire under C2):**
   - **superlative proper-noun collisions** — "**great**" in "**Great** Wall of
     China" fires `superlative_density`, which the gate deliberately leaves active
     so flattery is never softened. Pre-existing superlative-dependency limit,
     orthogonal to self-vs-other. (This case actually rises 0.99→1.00; see below.)
   - **low self-reference apologies** (`self_n < 2`) — a terse "I apologize for X.
     [third-person facts]" has one first-person token, so the gate stays
     conservative and does not neutralize; it relies on the word-boundary fix alone.
4. **Known interaction (flagged, not patched):** neutralizing `counter_lexicon`
   to its mean under `self_directed` removes a *legitimate* non-sycophancy signal
   when the self-directed text genuinely contains counter-vocabulary ("contrary",
   "however") — slightly *raising* its score. Rare (≈1 sample), did not breach any
   bar. A v1 refinement is to neutralize the counter/echo features **by omission
   only** (suppress the absence-inflation, keep a present counter word's protective
   effect). **Not changed here** — the frozen gate already cleared the bar; refining
   it post-hoc would be exactly the un-pre-registered tuning the discipline forbids.

## Ship recommendation (operator decision)

This cleared a real bar, so it is shippable in principle — but two parts, two paths:

- **Word-boundary tokenization fix (C1):** an unambiguous bug (`"fully" ∈
  "carefully"`). Ship into `sycophancy_signals._phrase_density` **after** re-running
  the v0 5-fold CV with word-boundary matching to confirm AUC holds (the published
  v0 calibration fingerprint was computed with substring matching).
- **Target gate (C2):** the harm is specifically that `needs_revision` fires on
  self-apology. The cleanest, lowest-risk integration is a **self-directed-register
  guard in `cognometrics._cogn_needs_revision`** — suppress the flag when the
  response is cleanly self-directed — mirroring 7.4.4's existing "only the gating
  decision is corrected, the instruments are not re-tuned" philosophy. This touches
  **no** published instrument weights or fingerprint. **Recommended gate before it
  ships: a cross-model replication** (the documented v1 priority), since this
  finding is in-distribution only.

**Decision taken this session:** report + validate + propose; do **not**
unilaterally edit live release-gating code, because (a) the evidence is
in-distribution only and project norm is cross-model replication before touching
published instruments, and (b) it changes release behavior. Integration is staged
as a reviewable proposal, not merged.
